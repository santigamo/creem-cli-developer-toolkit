## Codebase Patterns
- Keep the integration tiny: Bun runtime, Hono routes, and JSON file persistence instead of a database.
- Root workflow scripts load both `/.env` and `examples/minimal-integration/.env`; any env value with spaces must be quoted because the scripts use shell `source`.
- Capture reproducible evidence under `artifacts/` before recording; the AI prompts are designed to reason over those saved JSON files, not live terminal scrollback.
- Use the official `creem` SDK for API calls and keep webhook verification separate with manual HMAC over the raw body; the webhook route does not need the API key.
- Distributable Claude Code skills must live under `skills/<skill-name>/SKILL.md` with valid frontmatter; `npx skills add . --list` is the quickest local validation.
- Creem CLI money fields are minor units; amounts like `500 EUR` must be explained as `5.00 EUR`, not `500.00 EUR`.
